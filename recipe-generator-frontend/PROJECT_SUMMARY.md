## 技术实现

### 技术栈
- **Vue 3**: 使用 Composition API
- **Element Plus**: 完整的 UI 组件库
- **Vite**: 快速的构建工具
- **Axios**: HTTP 请求库

### 项目结构
```
recipe-generator-frontend/
├── src/
│   ├── components/
│   │   └── IngredientGrid.vue      # 食材网格组件
│   ├── views/
│   │   ├── HomePage.vue            # 首页（食材选择）
│   │   ├── RecipesPage.vue         # 菜谱展示页
│   │   ├── FavoritesPage.vue       # 收藏夹页
│   │   └── ShoppingPage.vue        # 购物清单页
│   ├── utils/
│   │   ├── api.js                  # API 接口
│   │   └── ingredientsData.js      # 食材数据
│   ├── App.vue                     # 根组件
│   ├── main.js                     # 入口文件
│   └── style.css                   # 全局样式
├── index.html
├── vite.config.js
├── package.json
└── README.md
```

### 核心功能

#### 1. 食材管理
- 70+ 种预设食材，带图标和默认用量
- 分类浏览和实时搜索
- 自定义添加食材
- 常用组合保存

#### 2. 智能筛选
- 多维度筛选（菜系、口味、时间、难度）
- 灵活组合筛选条件
- 与后端 API 集成

#### 3. 菜谱展示
- 响应式卡片布局
- 食材可用性标识
- 详细步骤展示
- 评分评论系统

#### 4. 数据持久化
使用 LocalStorage 存储：
- 当前菜谱
- 收藏列表
- 历史记录
- 评分评论
- 购物清单
- 常用组合

#### 5. 导出功能
- 菜谱导出为 TXT 文件
- 购物清单导出为 TXT 文件

## 启动说明

### 开发环境
```bash
cd recipe-generator-frontend
npm install
npm run dev
```
访问：http://localhost:3001

### 生产构建
```bash
npm run build
npm run preview
```

## API 集成

前端通过 Vite 代理连接后端：
- 开发环境：`/api` → `http://localhost:8000`
- 需要后端服务运行在 8000 端口
